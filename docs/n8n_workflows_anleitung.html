N8N Workflows einrichten

Schritt-für-Schritt Anleitung für alle E-Mail-Automatisierungen der BetreuerApp.

Was ist N8N? N8N ist eine Automatisierungs-Software. Sie empfängt Meldungen von der BetreuerApp (z.B. „neuer Betreuer registriert") und sendet daraufhin automatisch E-Mails. Sie müssen kein Programmierer sein – diese Anleitung führt Sie Schritt für Schritt durch alles.

🔧 Vorbereitung

Voraussetzungen: N8N muss bereits installiert und erreichbar sein (z.B. unter http://ihr-server:5678). Falls N8N noch nicht installiert ist, wenden Sie sich bitte an Ihren IT-Administrator.
1
N8N öffnen

Rufen Sie N8N in Ihrem Browser auf, z.B. http://ihr-server:5678 und melden Sie sich an.

2
E-Mail-Konto in N8N hinterlegen

Damit N8N E-Mails versenden kann, muss einmalig ein E-Mail-Konto eingerichtet werden:

  1. Klicken Sie oben rechts auf Ihren Namen → Settings
  2. Links auf Credentials → Add Credential
  3. Suchen Sie nach SMTP
  4. Tragen Sie die SMTP-Daten Ihres E-Mail-Anbieters ein (Fragen Sie ggf. Ihren IT-Admin)
  5. Klicken Sie auf Test → dann Save
Tipp: Bei Microsoft 365 / Exchange: Host = smtp.office365.com, Port = 587, STARTTLS aktivieren. Bei Google: smtp.gmail.com, Port = 465, SSL aktivieren.

🔗 BetreuerApp mit N8N verbinden

Einmalig muss die N8N-Adresse in der BetreuerApp eingetragen werden, damit die App weiß, wohin sie ihre Meldungen schicken soll.

1
Auf dem Server: Umgebungsvariable setzen

Öffnen Sie die Datei .env im Hauptverzeichnis der BetreuerApp und tragen Sie folgendes ein:

N8N_WEBHOOK_BASE_URL=http://ihr-n8n-server:5678

Ersetzen Sie ihr-n8n-server:5678 durch die tatsächliche Adresse Ihres N8N-Servers.

2
App neu starten

Nach der Änderung muss die BetreuerApp neu gestartet werden:

docker restart betreuer_django
Wie funktioniert das? Die BetreuerApp sendet bei bestimmten Aktionen automatisch eine Meldung an N8N (an die Adresse /webhook/betreuer-events). N8N empfängt diese Meldung und führt den passenden Workflow aus, z.B. eine E-Mail versenden.

📋 Übersicht: Welche Workflows müssen angelegt werden?

Es gibt einen einzigen Webhook (Empfangspunkt) in N8N. Der erste Schritt jedes Workflows prüft, welcher event_type angekommen ist und reagiert entsprechend. Sie können entweder einen großen Workflow mit mehreren Zweigen anlegen oder 5 separate Workflows. Diese Anleitung zeigt 5 separate Workflows – das ist für Nicht-Techniker übersichtlicher.

Workflow Wann wird er ausgelöst? E-Mail geht an
① Betreuer-Registrierung Wenn sich ein neuer Betreuer registriert Den neuen Betreuer (Passwort-Link)
② Stundennachweis genehmigt Wenn ein Nachweis genehmigt wird Buchhaltung (mit Betrag + PDF)
③ Freibetrag-Warnung Wenn ein Betreuer 80/90/100% des Freibetrags erreicht Admin / HR (Warnung)
④ Dokument läuft ab 30 Tage vor Ablauf eines Dokuments (täglich geprüft) Den Betreuer (Erinnerung)
⑤ Dokument abgelaufen Wenn ein Dokument abgelaufen ist Den Betreuer + Admin
Workflow ①

Betreuer-Registrierung

Wenn sich ein neuer Betreuer über den Registrierungslink anmeldet, erhält er automatisch eine E-Mail mit einem Link, um sein Passwort festzulegen.

🌐 Webhook
→
🔀 IF: event_type = betreuer_registered
→
✉️ E-Mail senden
1
Neuen Workflow erstellen
  1. Klicken Sie in N8N links oben auf + New Workflow
  2. Klicken Sie oben auf den Namen „My workflow" und benennen Sie ihn um: Betreuer Registrierung
2
Webhook-Node hinzufügen (Empfänger)
  1. Klicken Sie auf + Add first step oder das + Symbol
  2. Suchen Sie nach Webhook und wählen Sie es aus
  3. Setzen Sie folgende Einstellungen:
    HTTP Method: POST Path: betreuer-events Authentication: None
  4. Klicken Sie auf Listen for test event (für den Test später)
Wichtig: Den gleichen Webhook-Path (betreuer-events) verwenden alle 5 Workflows. N8N unterscheidet sie intern durch einen Filter im nächsten Schritt.
3
Filter-Node hinzufügen (nur den richtigen Event verarbeiten)
  1. Klicken Sie auf das + nach dem Webhook-Node
  2. Suchen Sie nach IF und wählen Sie es aus
  3. Konfigurieren Sie die Bedingung:
    Value 1: {{ "{{" }} $json.event_type {{ "}}" }} Operation: Equal Value 2: betreuer_registered
Erklärung: Der IF-Node schaut auf das Feld event_type in der eingehenden Meldung. Nur wenn es betreuer_registered ist, wird die E-Mail gesendet – sonst passiert nichts.
4
E-Mail-Node hinzufügen (am „True"-Ausgang des IF)
  1. Klicken Sie auf das + am grünen Ausgang des IF-Nodes (True)
  2. Suchen Sie nach Send Email (SMTP) und wählen Sie es aus
  3. Wählen Sie unter Credential Ihr zuvor eingerichtetes E-Mail-Konto
  4. Tragen Sie folgende Felder ein:
From: betreuerapp@csfv-minden.de To: {{ "{{" }} $json.betreuer_email {{ "}}" }} Subject: Willkommen bei der BetreuerApp – Bitte Passwort festlegen Body (HTML):
<p>Hallo {{ "{{" }} $json.betreuer_name {{ "}}" }},</p>
<p>willkommen bei der BetreuerApp des Christlichen Schulförderverein Minden e.V.</p>
<p>Sie wurden als Betreuer/in an der Schule <strong>{{ "{{" }} $json.school_name {{ "}}" }}</strong> registriert.</p>
<p>Bitte klicken Sie auf den folgenden Link, um Ihr Passwort festzulegen:</p>
<p><a href="{{ "{{" }} $json.password_reset_url {{ "}}" }}">Passwort festlegen</a></p>
<p><em>Dieser Link ist 3 Tage gültig.</em></p>
<p>Mit freundlichen Grüßen<br>CSFV Minden</p>
Verfügbare Felder aus der App: betreuer_name, betreuer_email, school_name, school_code, contract_number, password_reset_url
5
Workflow aktivieren und speichern
  1. Klicken Sie oben rechts auf Save
  2. Klicken Sie auf den Schalter Inactive → Active oben rechts
Workflow ②

Stundennachweis genehmigt

Wenn ein Koordinator einen Stundennachweis genehmigt, erhält die Buchhaltung automatisch alle abrechnungsrelevanten Daten (Stunden, Betrag, Projektnummer, Kreditorennummer, PDF-Link).

🌐 Webhook
→
🔀 IF: timesheet_approved
→
✉️ E-Mail an Buchhaltung
Genauso wie Workflow ①, nur mit diesen Abweichungen:
Workflow-Name: Stundennachweis genehmigt
IF-Bedingung (Value 2): timesheet_approved
E-Mail-Einstellungen:
To: buchhaltung@csfv-minden.de ← feste Adresse der Buchhaltung Subject: Abrechnung {{ "{{" }} $json.betreuer_name {{ "}}" }} – {{ "{{" }} $json.month {{ "}}" }}/{{ "{{" }} $json.year {{ "}}" }}
E-Mail-Text (HTML):
<p>Ein Stundennachweis wurde genehmigt und steht zur Auszahlung bereit:</p>
<table>
  <tr><td><strong>Betreuer/in:</strong></td><td>{{ "{{" }} $json.betreuer_name {{ "}}" }}</td></tr>
  <tr><td><strong>Schule:</strong></td><td>{{ "{{" }} $json.school_name {{ "}}" }}</td></tr>
  <tr><td><strong>Monat:</strong></td><td>{{ "{{" }} $json.month {{ "}}" }}/{{ "{{" }} $json.year {{ "}}" }}</td></tr>
  <tr><td><strong>Stunden:</strong></td><td>{{ "{{" }} $json.total_hours {{ "}}" }} h</td></tr>
  <tr><td><strong>Betrag:</strong></td><td>{{ "{{" }} $json.total_amount {{ "}}" }} €</td></tr>
  <tr><td><strong>Projektnummer:</strong></td><td>{{ "{{" }} $json.projektnummer {{ "}}" }}</td></tr>
  <tr><td><strong>Kreditorennummer:</strong></td><td>{{ "{{" }} $json.kreditorennummer {{ "}}" }}</td></tr>
  <tr><td><strong>Vertragsnummer:</strong></td><td>{{ "{{" }} $json.contract_number {{ "}}" }}</td></tr>
</table>
<p>PDF: <a href="{{ "{{" }} $json.pdf_url {{ "}}" }}">Stundennachweis herunterladen</a></p>
Verfügbare Felder: betreuer_name, betreuer_email, contract_number, school_name, school_code, month, year, total_hours, total_amount, projektnummer, kreditorennummer, pdf_url
Workflow ③

Freibetrag-Warnung

Wenn ein Betreuer 80%, 90% oder 100% des Freibetrags (3.300 €/Jahr) erreicht, erhält die Verwaltung eine Warnung. So kann frühzeitig reagiert werden.

🌐 Webhook
→
🔀 IF: freibetrag_warning
→
✉️ E-Mail an Admin/HR
Abweichungen zu Workflow ①:
Workflow-Name: Freibetrag Warnung
IF-Bedingung: freibetrag_warning
To: personal@csfv-minden.de ← Personalstelle / HR Subject: ⚠️ Freibetrag-Warnung: {{ "{{" }} $json.betreuer_name {{ "}}" }} ({{ "{{" }} $json.warning_level {{ "}}" }})
E-Mail-Text (HTML):
<p>Der Freibetrag eines Betreuers hat eine Warnstufe erreicht:</p>
<table>
  <tr><td><strong>Betreuer/in:</strong></td><td>{{ "{{" }} $json.betreuer_name {{ "}}" }}</td></tr>
  <tr><td><strong>Jahr:</strong></td><td>{{ "{{" }} $json.year {{ "}}" }}</td></tr>
  <tr><td><strong>Verbraucht:</strong></td><td>{{ "{{" }} $json.total_used {{ "}}" }} € ({{ "{{" }} $json.percentage {{ "}}" }}%)</tr>
  <tr><td><strong>Freibetrag:</strong></td><td>{{ "{{" }} $json.limit {{ "}}" }} €</td></tr>
  <tr><td><strong>Verbleibend:</strong></td><td>{{ "{{" }} $json.remaining {{ "}}" }} €</td></tr>
  <tr><td><strong>Warnstufe:</strong></td><td>{{ "{{" }} $json.warning_level {{ "}}" }}</td></tr>
</table>
<p>Bitte prüfen Sie die Stundenplanung für diesen Betreuer.</p>
Warnstufen: yellow = 80% erreicht · orange = 90% erreicht · red = 100% überschritten
Verfügbare Felder: betreuer_name, betreuer_email, year, percentage, total_used, remaining, limit, warning_level
Workflow ④

Dokument läuft ab

30 Tage vor Ablauf eines Pflichtdokuments (z.B. Führungszeugnis) erhält der Betreuer eine Erinnerungs-E-Mail. Diese Prüfung läuft automatisch jeden Tag.

🌐 Webhook
→
🔀 IF: document_expiring
→
✉️ E-Mail an Betreuer
Abweichungen zu Workflow ①:
Workflow-Name: Dokument läuft ab
IF-Bedingung: document_expiring
To: {{ "{{" }} $json.betreuer_email {{ "}}" }} Subject: ⏰ Erinnerung: Ihr {{ "{{" }} $json.document_type {{ "}}" }} läuft in {{ "{{" }} $json.days_remaining {{ "}}" }} Tagen ab
E-Mail-Text (HTML):
<p>Hallo {{ "{{" }} $json.betreuer_name {{ "}}" }},</p>
<p>Ihr Dokument <strong>{{ "{{" }} $json.document_type {{ "}}" }}</strong> läuft in
<strong>{{ "{{" }} $json.days_remaining {{ "}}" }} Tagen</strong> ({{ "{{" }} $json.expires_at {{ "}}" }}) ab.</p>
<p>Bitte erneuern Sie das Dokument rechtzeitig und laden Sie es in der BetreuerApp hoch.</p>
<p>Falls Sie Fragen haben, wenden Sie sich an Ihren Koordinator.</p>
<p>Mit freundlichen Grüßen<br>CSFV Minden</p>
Verfügbare Felder: betreuer_name, betreuer_email, document_type, expires_at, days_remaining
Workflow ⑤

Dokument abgelaufen

Wenn ein Pflichtdokument abgelaufen ist, werden sowohl der Betreuer als auch die Verwaltung benachrichtigt.

🌐 Webhook
→
🔀 IF: document_expired
→
✉️ E-Mail an Betreuer
+
✉️ E-Mail an Admin
Abweichungen zu Workflow ①:
Workflow-Name: Dokument abgelaufen
IF-Bedingung: document_expired
Zwei E-Mails: Nach dem IF-Node fügen Sie zwei E-Mail-Nodes parallel ein: einen für den Betreuer und einen für die Verwaltung. Verbinden Sie dazu den IF-True-Ausgang mit beiden E-Mail-Nodes.
E-Mail 1 – An Betreuer:
To: {{ "{{" }} $json.betreuer_email {{ "}}" }} Subject: ❌ Ihr {{ "{{" }} $json.document_type {{ "}}" }} ist abgelaufen
<p>Hallo {{ "{{" }} $json.betreuer_name {{ "}}" }},</p>
<p>Ihr Dokument <strong>{{ "{{" }} $json.document_type {{ "}}" }}</strong> ist am {{ "{{" }} $json.expired_at {{ "}}" }} abgelaufen.</p>
<p>Bitte erneuern Sie das Dokument umgehend und laden Sie es in der BetreuerApp hoch.</p>
<p>Ohne gültige Dokumente können Sie leider nicht weiter eingesetzt werden.</p>
E-Mail 2 – An Verwaltung:
To: personal@csfv-minden.de Subject: ❌ Abgelaufenes Dokument: {{ "{{" }} $json.betreuer_name {{ "}}" }}
<p>Das folgende Dokument ist abgelaufen:</p>
<ul>
  <li><strong>Betreuer/in:</strong> {{ "{{" }} $json.betreuer_name {{ "}}" }} ({{ "{{" }} $json.betreuer_email {{ "}}" }})</li>
  <li><strong>Dokument:</strong> {{ "{{" }} $json.document_type {{ "}}" }}</li>
  <li><strong>Abgelaufen am:</strong> {{ "{{" }} $json.expired_at {{ "}}" }}</li>
</ul>
<p>Bitte kontaktieren Sie den Betreuer und stellen Sie sicher, dass das Dokument erneuert wird.</p>
Verfügbare Felder: betreuer_name, betreuer_email, document_type, expired_at

✅ Workflows testen

So testen Sie, ob alles funktioniert, ohne echte Daten zu verändern:

Test: Betreuer-Registrierung (Workflow ①)
  1. Öffnen Sie in N8N den Workflow „Betreuer Registrierung"
  2. Klicken Sie auf den Webhook-Node → Listen for test event
  3. Öffnen Sie ein Terminal und geben Sie folgenden Befehl ein:
curl -X POST http://ihr-n8n-server:5678/webhook-test/betreuer-events \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "betreuer_registered",
    "betreuer_name": "Erika Testperson",
    "betreuer_email": "ihre-email@beispiel.de",
    "school_name": "Grundschule Testschule",
    "school_code": "GST",
    "contract_number": "VN-2026-001",
    "password_reset_url": "http://betreuer-app.de/accounts/passwort-setzen/test/test/"
  }'
Tipp: Ersetzen Sie ihre-email@beispiel.de mit Ihrer eigenen E-Mail-Adresse. So sehen Sie die Test-E-Mail direkt in Ihrem Postfach.
Einfacherer Test: Direkt aus der BetreuerApp
  1. Melden Sie sich als Admin in der BetreuerApp an
  2. Erstellen Sie einen Registrierungslink für eine Testschule
  3. Öffnen Sie den Registrierungslink und registrieren Sie sich mit einer Test-E-Mail-Adresse
  4. Prüfen Sie das E-Mail-Postfach – die Willkommens-E-Mail sollte ankommen
Test: Stundennachweis genehmigt (Workflow ②)
  1. Melden Sie sich als Betreuer an und erfassen Sie einen Stundeneintrag
  2. Reichen Sie den Stundennachweis ein (Button „Einreichen")
  3. Melden Sie sich als Koordinator an und genehmigen Sie den Nachweis
  4. Prüfen Sie das E-Mail-Postfach der Buchhaltung

❓ Häufige Probleme

N8N empfängt keine Meldungen von der App
  • Ist N8N_WEBHOOK_BASE_URL korrekt in der .env-Datei gesetzt?
  • Wurde die BetreuerApp nach der Änderung neu gestartet? (docker restart betreuer_django)
  • Ist der N8N-Server vom BetreuerApp-Server aus erreichbar? (Firewall prüfen)
  • Ist der Workflow in N8N auf Active gestellt?
E-Mail kommt nicht an
  • Prüfen Sie den Spam-Ordner
  • Testen Sie das SMTP-Konto in N8N (Settings → Credentials → Test)
  • Schauen Sie in den N8N-Ausführungsprotokollen nach (Klick auf Executions)
Falsche E-Mail-Adresse als Empfänger
  • Prüfen Sie im E-Mail-Node das „To:"-Feld
  • Für Betreuer-E-Mails: {{ "{{" }} $json.betreuer_email {{ "}}" }} (kommt aus der App)
  • Für feste Empfänger (Buchhaltung, HR): direkte E-Mail-Adresse eintragen
